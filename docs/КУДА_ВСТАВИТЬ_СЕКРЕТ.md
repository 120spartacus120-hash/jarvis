# Куда вставить секрет на GitHub (2 минуты)

## Ссылка

Открой: **https://github.com/120spartacus120-hash/jarvis/settings/secrets/actions**

(или: репозиторий **jarvis** → вкладка **Settings** → слева **Secrets and variables** → **Actions**)

## Шаги

1. Нажми зелёную кнопку **New repository secret**
2. **Name** (имя): скопируй точно:
   ```
   TAURI_SIGNING_PRIVATE_KEY
   ```
3. **Secret** (значение):
   - Открой на компьютере файл:
     ```
     C:\Users\Asus\Desktop\EXE\Jarvis\jarvis-master\.tauri\jarvis-updater.key
     ```
   - Блокнотом → **Ctrl+A** → **Ctrl+C** (вся одна длинная строка)
   - Вставь в поле **Secret** на GitHub
4. Нажми **Add secret**

## Что НЕ делать

- Не создавай секрет `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` — не нужен
- Не отправляй файл `.key` никому в чат — только в это поле на GitHub
- Файл `.tauri/jarvis-updater.key.pub` — это публичный ключ, его в секрет не клади

## После секрета

Если сборка в **Actions** уже упала без секрета:
1. **Actions** → workflow **Release** → последний запуск → **Re-run all jobs**

Когда сборка зелёная — релиз здесь:  
**https://github.com/120spartacus120-hash/jarvis/releases**
